
# 🗂 タスク管理ツール：アーカイブ機能 仕様書（Copilot 用）

## 🎯 概要
本ツールは **ツリー構造のタスク管理アプリ**であり、  
**1つの管理ファイルで全タスクを保持**する。

アーカイブ機能では、  
選択したタスクを **アーカイブ専用ファイルへ移動**し、  
復元可能な状態で保管する。

---

# 📦 アーカイブ機能の設計方針

## 1. アーカイブ動作
- タスクアイテムをアーカイブすると、以下をアーカイブファイルへ移動する  
  - タスク内容（タイトル、説明、属性など）  
  - タスクのツリーパス（親子構造を示すパス）  
  - アーカイブ実行日（`ArchivedAt`）

- アーカイブ後、元の管理ファイルからは該当タスクを削除する。

### アーカイブ時に保持する情報（例）
```json
{
  "Name": "xxx",
  "Path": "Root/ProjectA",
  "Memo": "...",
  "Link": "https://example.com",
  "BeginDate": "2026-04-01",
  "EndDate": "2026-04-10",
  "Completed": false,
  "IsWait": false,
  "MarkLevel": 1,
  "ArchivedAt": "2026-04-10T23:00:00"
}
```

---

## 2. アーカイブファイルの構造
- アーカイブ専用ファイルは **通常のタスク管理ファイルと同じファイル名に「.archive」を付加**して管理する。
- ただし、アーカイブされたタスクは **ツリー構造を維持しない**  
  → 復元時にパスを参照するため、**フラットなリスト**として保持する。

---

# 🔄 復元（アンアーカイブ）機能

## 1. 復元 UI（ゴミ箱方式）
- アーカイブファイル内のタスクを一覧表示するウィンドウを提供する。
- 表示項目：
  - タスク名
  - 元のパス
  - アーカイブ日
- ユーザーはチェックボックスで復元対象を選択する。

## 2. 復元動作
- 選択されたタスクを **元のパスに再配置**する。
- 元のパスが存在しない場合は、**不足する親フォルダを自動作成して復元**する。

- 復元後、アーカイブファイルから該当タスクを削除する。

---

# 🧱 データ構造

データ構造は基本的にお任せします。

- タスク管理ファイルとアーカイブファイルは同一のクラス構造で管理してもよい。


# ⚙️ 処理フロー

## アーカイブ処理
1. ユーザーがタスクを選択し「アーカイブ」を実行  
2. タスク内容＋パス＋アーカイブ日をアーカイブファイルへ追加  
3. 元の管理ファイルからタスクを削除  
4. ファイルを保存

## 復元処理
1. アーカイブ一覧ウィンドウを表示  
2. 復元対象を選択  
3. 元のパスへタスクを再配置  
4. アーカイブファイルから削除  
5. 管理ファイルを保存

---

# 🧭 Copilot に理解させたいポイント（重要）

- **タスクは 1 ファイルで管理し、アーカイブは別ファイルに移動する**  
- **アーカイブ時にパスと日付を必ず保存する**  
- **復元は「元のパスに戻す」ことが基本動作**  
- **復元 UI はゴミ箱方式で一覧から選択する**  
- **アーカイブファイルはフラット構造でよい**  
- **復元時にパスが存在しない場合、親ツリーを含め再現させる**
